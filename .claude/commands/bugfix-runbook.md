# Bugfix Runbook

バグ修正時に闇雲に触らせない。再現→原因特定→最小修正→回帰確認の順で進める。

## Steps

1. **再現確認**: バグの症状を具体的に確認する
2. **原因仮説**: 可能性のある原因を列挙する（最低3つ）
3. **原因特定**: コードを読んで最も可能性が高い原因を特定する
4. **最小修正**: 影響範囲を最小限にした修正を行う
5. **回帰確認**: `flutter test` 全パス / `flutter analyze` 0 issues を確認する
6. **報告**: 修正内容と根本原因をまとめる

## Investigation Checklist

- [ ] エラーメッセージを完全に読んだか
- [ ] スタックトレースで該当ファイル・行を特定したか
- [ ] 最近の変更で影響があったファイルはないか
- [ ] `if (mounted)` 漏れはないか
- [ ] dispose漏れはないか
- [ ] null安全が守られているか
- [ ] 非同期処理の順序は正しいか

## Rules
- 症状を消すだけの修正（`// ignore` 等）はしない
- 修正範囲は必要最小限にする
- 修正後は必ず回帰テストを実行する

## Output Format

```markdown
## バグ概要
[症状の説明]

## 根本原因
[なぜ起きたか]

## 修正内容
- [lib/xxx.dart:行番号] — [変更内容]

## flutter test 結果（修正後）
[全パス / X件失敗]

## flutter analyze 結果（修正後）
[0 issues / X issues]

## 再発防止策
[同種バグを防ぐために追加したテスト等]
```
