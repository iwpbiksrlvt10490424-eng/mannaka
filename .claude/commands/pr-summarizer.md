# PR Summarizer

夜に帰宅したユーザーがレビューしやすい形で変更内容をまとめる。

## Steps

1. 今日の変更ファイルを確認する（git diff または実装メモを読む）
2. `~/mannaka/team/workspace/` の各エージェント出力を読む
3. 以下のフォーマットで日本語サマリーを作成する

## Output Format

```markdown
# 本日の開発サマリー

## 今日の成果（2〜4行）
[何を実装・修正・改善したか、ユーザー視点で説明]

## 変更ファイル一覧
- lib/xxx.dart — [変更内容]
- test/xxx_test.dart — [テスト内容]

## なぜこのアプローチか
[設計判断の理由。後から読んでも分かるように]

## 検証結果
- flutter analyze: [0 issues / X issues]
- flutter test: [全パス / X件失敗]

## 残課題・リスク
- [課題1]: [対処が必要な理由]

## 明日の最優先タスク（1つ）
[次に何をすべきか]

## 開発者へのメッセージ
[注意点・確認してほしいこと・判断が必要な事項]
```

## Rules
- 技術用語を使う場合は文脈を添える
- 変更の「なぜ」を必ず書く
- 判断が必要な事項はUser Inputとして明示する
- 長すぎない（全体で20行以内を目安）
